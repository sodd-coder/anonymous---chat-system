The server never computes sharedRoomKey. It only ever sees opaque ciphertext. Even with full server access, messages are unreadable.
⚡
X25519 Key Exchange
Elliptic Curve Diffie-Hellman. Both sides derive the same shared secret without ever transmitting it. A wiretapper on the server connection learns nothing.
🔒
XSalsa20-Poly1305
Authenticated encryption. Each message gets a random nonce. Poly1305 MAC ensures nobody tampered with the ciphertext in transit — integrity guaranteed.
🌀
Ephemeral Keys Per Session
Keypairs generated fresh every session, discarded on close. Past sessions are unrecoverable even if a future key were somehow compromised.
✅
TweetNaCl is Audited
The underlying library has been independently security-audited. It's the same cryptography family used in Signal and dozens of high-security applications worldwide.

No — mathematically impossible. Messages are encrypted with XSalsa20-Poly1305 using a shared key derived via X25519 ECDH. That key is never transmitted to the server. The server only sees opaque base64 ciphertext. Even with full database access, messages are unreadable.
When the last user leaves, the room object is immediately deleted from server memory. Messages were never written to a database — they exist only in runtime memory during the session. The ephemeral keypairs are discarded. There is no recovery path.
Your IP is used to derive a device hash for abuse prevention (rate limiting, reputation) but is never stored after the session ends. No analytics, no tracking pixels, no third-party scripts in the chat interface.
The room creator generates a random 32-byte room session key. When a user joins and shares their public key, the creator encrypts the room key for that user using ECDH box (nacl.box). The server relays this encrypted blob but cannot open it — only the recipient's secret key can decrypt it.
TweetNaCl.js — a port of Daniel J. Bernstein's NaCl library. It's independently security audited, used in production by Signal and others, and implements X25519 + XSalsa20-Poly1305 authenticated encryption. No proprietary crypto.
